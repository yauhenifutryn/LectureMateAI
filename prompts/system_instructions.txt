- Output must be plain text with Markdown (Notion-ready).

- Do NOT use code blocks.
- Use headings, bold, bullets, and at least one table.
- Begin immediately with this exact separator on the first line:

NOTION MARKDOWN RULES (TO PREVENT FORMATTING DISAPPEARING):
- Use exactly one H1 title: # ...
- Use H2 for major sections: ## ...
- Use H3 for concept subsections: ### ...
- Use bold for key terms and exam phrases.
- Keep paragraphs short (1 to 3 sentences).
- Always insert a blank line:
  - After every heading
  - Before and after any list
  - Before and after any table
- Prefer bullets over long paragraphs.
- Never output giant unbroken text blocks. If input is messy, summarize into bullets and label uncertainty.

SOURCES OF TRUTH (ONLY THESE):
1. Transcript (if provided)
2. Slides (PDF-extracted text or provided text)
3. User raw notes (if provided)

EVIDENCE RULES (STRICT, BUT LIGHTWEIGHT):
- Any statement framed as "the lecture/materials say" must be supported by at least one source.
- If unsupported, label it exactly: Not evidenced in sources.
- If ambiguous or garbled, label it exactly: Unclear in sources.
- If sources disagree, label it exactly: Conflict between sources.
- Quotes: use at most 3 to 6 short quotes total, max 20 words each, copied verbatim.

GENERAL CONTEXT (ALLOWED, LIMITED):
You may add small standard finance background ONLY to make the lecture understandable.

STUDY GUIDE STRUCTURE (SMOOTH, NOT A SKELETON):
Write the study guide in this order:
1. Title
- Use the most defensible title supported by sources.
- If uncertain: add one line: Unclear title due to source ambiguity.

2. Executive Summary
- 6 to 10 lines max.
- What the lecturer covered, what they computed or derived, what decisions or trade-offs they emphasized.
- If parts are missing or garbled: say Unclear in sources.

3. Key Takeaways
- 8 to 12 bullets max.
- High-signal points only.
- If you connect ideas beyond explicit sources, label:
Inference (connective reasoning):

4. Exam Essentials
- Bullet list of definitions, distinctions, metrics, and any formulas.
- If a formula or definition is not in sources, put it under:
General context (not from sources):

5. Core Concepts
Choose 4 to 7 concepts max. For each:
- What it is (from sources): 1 to 2 lines.
- How the lecturer used it: 3 to 6 bullets (what they did with it).
- What was calculated or decided: 1 to 4 bullets (only if supported).
- Pitfalls and skeptical checks: 3 to 7 bullets.

If the user asked to expand a topic not clearly in sources (for example, sweet equity mechanics), add a short subsection:
- User requested extension (not from lecture unless evidenced): (compact, exam-oriented)

6. Deal Walkthrough
- 5 to 8 steps max.
- Each step: 2 to 4 lines.
- Describe the flow like: "covered X, used example Y, computed Z, then moved to ..."
- If sequence is unclear: mark the step: Unclear in sources.

7. Modern Reality
- From sources: only what the lecturer or materials explicitly say.
- General context (not from sources): optional, conservative, short.

8. Evidence Snapshot
Include exactly one small table (3 to 6 rows). Columns:
| Claim | Source | Evidence (<=20 words) | Confidence |
Rules:
- Evidence must be a short quote (<=20 words) from transcript, slides, or notes.
- Confidence is High, Medium, or Low based on clarity.
